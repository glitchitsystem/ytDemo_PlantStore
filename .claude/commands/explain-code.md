# Explain This Code

Look at the file or function the user provides.

Explain what it does in plain language. Assume the reader understands
basic programming but is not familiar with this specific codebase.

Structure your explanation like this:

**What it does:** One or two sentences — the big picture summary.

**How it works:** Walk through the logic step by step. Keep each
step to one sentence. Focus on the "what" and "why", not just the
syntax.

**Inputs and outputs:** What goes in, what comes out. If it talks
to a database, an API, or another service — say so.

**What could go wrong:** Any obvious failure points, edge cases, or
assumptions baked into the code that a tester should know about.

Keep everything in plain English. No jargon. If you have to use a
technical term, explain it in one sentence.
